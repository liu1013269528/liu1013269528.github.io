命令概览 ​

用法 ​

bash
nanobot gateway [OPTIONS]

选项 ​

选项简写默认值说明
--port-p18790Gateway 端口（预留）
--workspace-w从配置读取工作空间目录
--config-c~/.nanobot/config.json配置文件路径
--verbose-vfalse详细输出

功能 ​

gateway 命令启动 nanobot 的网关服务，包括：

  1. 加载配置
  2. 初始化所有核心组件
  3. 启动聊天通道
  4. 启动后台服务
  5. 运行消息处理循环

1. MessageBus (消息总线) ​

文件: nanobot/bus/queue.py

python
class MessageBus:
    """
    消息总线 - 解耦通道和代理
    """
    def __init__(self):
        # 入站队列：聊天平台 → Agent
        self.inbound = asyncio.Queue()

        # 出站队列：Agent → 聊天平台
        self.outbound = asyncio.Queue()

初始化时机：gateway 启动时立即创建

作用：

  • 解耦各组件
  • 提供异步消息传递
  • 缓冲消息避免阻塞

2. LLM Provider (LLM 提供商) ​

文件: nanobot/providers/registry.py

python
def _make_provider(config: Config):
    """
    根据配置创建对应的 LLM 提供商

    匹配规则：
    1. 检查 model 前缀 (claude- → anthropic)
    2. 查找 PROVIDERS 注册表
    3. 创建对应的 Provider 实例
    """
    model = config.agents.defaults.model
    provider_name = config.get_provider_name(model)
    provider_config = config.get_provider(model)

    # 创建提供商实例
    if provider_name == "anthropic":
        return AnthropicProvider(provider_config)
    elif provider_name == "openai":
        return OpenAIProvider(provider_config)
    else:
        return LiteLLMProvider(provider_config)

3. SessionManager (会话管理器) ​

文件: nanobot/session/manager.py

python
class SessionManager:
    """
    会话管理器 - 管理对话历史
    """
    def __init__(self, workspace: Path):
        self.workspace = workspace
        self.sessions_dir = workspace / "sessions"
        self._cache = {}  # 内存缓存

    def get_or_create(self, key: str) -> Session:
        """
        获取或创建会话

        key 格式：channel:chat_id
        例如：telegram:123456789
        """
        if key in self._cache:
            return self._cache[key]

        # 从磁盘加载或创建新会话
        session = self._load(key) or Session(key=key)
        self._cache[key] = session
        return session

会话存储格式：JSONL (每行一个 JSON)

{"role": "user", "content": "你好"}
{"role": "assistant", "content": "你好！有什么我可以帮助你的？"}

4. CronService (定时任务服务) ​

文件: nanobot/cron/service.py

python
class CronService:
    """
    定时任务服务
    """
    def __init__(self, store_path: Path):
        self.store_path = store_path
        self.jobs = {}  # 任务字典

    async def start(self):
        """启动定时任务调度器"""
        # 加载已保存的任务
        self._load_jobs()

        # 启动调度循环
        asyncio.create_task(self._scheduler_loop())

    async def _scheduler_loop(self):
        """调度循环：每秒检查一次"""
        while True:
            await asyncio.sleep(1)
            await self._check_and_run_jobs()

任务类型：

类型说明示例

every间隔重复每 5 分钟执行一次
cronCron 表达式每天 9:00 执行

5. HeartbeatService (心跳服务) ​

文件: nanobot/heartbeat/service.py

python
class HeartbeatService:
    """
    心跳服务 - 定期触发任务
    """
    async def start(self, callback, agent):
        """
        启动心跳

        callback: 触发时的回调函数
        agent: Agent 实例，用于执行任务
        """
        # 加载 HEARTBEAT.md
        tasks = self._load_heartbeat_tasks()

        # 计算下次触发时间
        next_time = self._calculate_next_time()

        # 启动定时器
        while True:
            await asyncio.sleep(next_time - now)
            await callback(agent, tasks)
            next_time = self._calculate_next_time()

6. ChannelManager (通道管理器) ​

文件: nanobot/channels/manager.py

python
class ChannelManager:
    """
    通道管理器 - 管理所有聊天通道
    """
    def __init__(self, config: Config, bus: MessageBus):
        self.config = config
        self.bus = bus
        self.channels = []

    async def start(self):
        """启动所有启用的通道"""
        for name in self.config.enabled:
            channel = self._create_channel(name)
            await channel.start()
            self.channels.append(channel)

    def _create_channel(self, name: str):
        """创建通道实例"""
        if name == "telegram":
            return TelegramChannel(
                config=self.config.telegram,
                bus=self.bus
            )
        elif name == "discord":
            return DiscordChannel(
                config=self.config.discord,
                bus=self.bus
            )

7. AgentLoop (代理循环) ​

文件: nanobot/agent/loop.py

python
class AgentLoop:
    """
    代理循环 - 消息处理的核心引擎
    """
    def __init__(
        self,
        config: Config,
        bus: MessageBus,
        provider: BaseProvider,
        session_manager: SessionManager,
        cron: CronService,
        heartbeat: HeartbeatService
    ):
        self.config = config
        self.bus = bus
        self.provider = provider
        self.session_manager = session_manager
        self.cron = cron
        self.heartbeat = heartbeat

        # 注册默认工具
        self._register_default_tools()

    async def run(self):
        """主循环"""
        while True:
            # 从 inbound 队列获取消息
            msg = await self.bus.consume_inbound()

            # 分发并处理消息
            await self._dispatch(msg)

运行时行为 ​

消息处理流程 ​

用户在 Telegram 发送消息
         │
         ▼
    TelegramChannel 收到消息
         │
         ├─► 检查权限 (is_allowed)
         ├─► 下载媒体（如果有）
         └─► 创建 InboundMessage
         │
         ▼
    bus.inbound.put(msg)
         │
         ▼
    AgentLoop.run() 消费消息
         │
         ├─► 获取会话 (SessionManager)
         ├─► 构建上下文 (ContextBuilder)
         ├─► 调用 LLM (Provider)
         ├─► 执行工具 (ToolRegistry)
         ├─► 保存会话
         └─► 发送响应 → bus.outbound
         │
         ▼
    TelegramChannel.send()
         │
         ▼
    用户收到回复

async def _shutdown(signum, frame):
    """优雅关闭"""
    # 1. 停止通道
    await channel_manager.stop()

    # 2. 停止服务
    await cron.stop()
    await heartbeat.stop()

    # 3. 保存会话
    session_manager.save_all()

    # 4. 退出
    sys.exit(0)

常见问题 ​

Q: Gateway 启动失败？ ​

A: 检查：

  1. 配置文件是否存在
  2. API Key 是否正确
  3. 通道 token 是否正确
  4. 网络连接是否正常

Q: 通道无法连接？ ​

A:

  1. 检查 enabled 配置
  2. 检查通道 token
  3. 查看日志信息

Q: 如何运行多个实例？ ​

A: 使用不同的配置文件

bash
nanobot gateway --config ~/.nanobot-telegram/config.json
nanobot gateway --config ~/.nanobot-discord/config.json
