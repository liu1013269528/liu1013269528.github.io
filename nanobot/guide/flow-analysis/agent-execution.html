AgentLoop 概述 ​

职责 ​

AgentLoop 是 nanobot 的核心处理引擎，负责：

  1. 从 MessageBus 接收消息
  2. 构建完整的 LLM 上下文
  3. 调用 LLM 获取响应
  4. 执行工具调用
  5. 保存对话历史
  6. 发送响应

生活类比 ​

AgentLoop 就像是"大脑"的思考过程：

┌─────────────────────────────────────────────────┐
│              AgentLoop (大脑)                   │
├─────────────────────────────────────────────────┤
│                                                  │
│  👂 接收信息 → 🧠 思考 → 🗣️ 回答               │

主循环结构 ​

文件: nanobot/agent/loop.py

python
class AgentLoop:
    async def run(self):
        """
        主循环：持续从 inbound 队列消费消息
        """
        while True:
            # 1. 从队列获取消息（阻塞等待）
            msg = await self.bus.consume_inbound()

            # 2. 分发处理消息
            await self._dispatch(msg)

消息处理 ​

_dispatch 方法 ​

python
async def _dispatch(self, msg: InboundMessage):
    """
    分发消息到相应的处理器
    """
    # 处理斜杠命令
    if msg.content.startswith("/"):
        await self._handle_command(msg)
        return

    # 处理普通消息
    await self._process_message(msg)

斜杠命令 ​

命令功能

/stop停止当前会话的所有任务
/help显示帮助信息

_process_message 方法 ​

python
async def _process_message(self, msg: InboundMessage):
    """
    处理用户消息
    """
    # 1. 获取会话
    session_key = msg.session_key or f"{msg.channel}:{msg.chat_id}"
    session = self.session_manager.get_or_create(session_key)

    # 2. 检查会话是否已停止
    if session.stopped:
        await self._send_stopped_message(msg)
        return

    # 3. 构建消息列表
    messages = await self._build_messages(msg, session)

    # 4. 运行 Agent 循环
    result = await self._run_agent_loop(messages)

    # 5. 保存会话
    session.add_message({
        "role": "user",
        "content": msg.content,
        "media": msg.media
    })
    session.add_message({
        "role": "assistant",
        "content": result.content
    })

    # 6. 检查是否需要记忆合并
    if len(session.messages) > self.config.memory_window:
        await self._consolidate_session(session)

    # 7. 发送响应
    await self._send_response(msg, result)

Agent 迭代循环 ​

_run_agent_loop 方法 ​

这是 Agent 的核心迭代逻辑，处理 LLM 和工具的交互循环。

python
async def _run_agent_loop(
    self,
    messages: list[dict],
    max_iterations: int = 40
) -> AgentResult:
    """
    Agent 迭代循环：LLM → 工具执行 → LLM → ...

    最大迭代次数：40（防止无限循环）
    """
    iteration = 0
    tool_results = []

    while iteration < max_iterations:
        iteration += 1

        # 1. 调用 LLM
        response = await self.provider.chat(
            messages=messages,
            tools=self.tool_registry.get_definitions()
        )

        # 2. 检查是否有工具调用
        if not response.tool_calls:
            # 没有工具调用，返回最终内容
            return AgentResult(
                content=response.content,
                tool_calls=[],
                iterations=iteration
            )

        # 3. 执行所有工具调用
        for tool_call in response.tool_calls:
            result = await self.tool_registry.execute(
                tool_call.name,
                tool_call.arguments
            )

            # 记录工具结果
            tool_results.append({
                "tool_call_id": tool_call.id,
                "role": "tool",
                "name": tool_call.name,
                "content": result
            })

        # 4. 将工具结果添加到消息
        messages.extend(tool_results)

    # 达到最大迭代次数
    return AgentResult(
        content="达到最大迭代次数，请简化您的请求",
        iterations=max_iterations
    )

上下文构建 ​

_build_messages 方法 ​

python
async def _build_messages(
    self,
    msg: InboundMessage,
    session: Session
) -> list[dict]:
    """
    构建发送给 LLM 的完整消息列表
    """
    messages = []

    # 1. 构建系统提示词
    system_prompt = await self.context_builder.build(
        config=self.config,
        session=session
    )
    messages.append({
        "role": "system",
        "content": system_prompt
    })

    # 2. 添加历史对话
    history = session.get_history(max_messages=100)
    messages.extend(history)

    # 3. 添加当前消息
    messages.append({
        "role": "user",
        "content": msg.content,
        "media": msg.media  # 如果有图片/文件
    })

    return messages

工具执行 ​

ToolRegistry ​

文件: nanobot/agent/tools/registry.py

python
class ToolRegistry:
    """
    工具注册表 - 管理所有可用工具
    """
    def __init__(self):
        self._tools = {}

    def register(self, tool: BaseTool):
        """注册工具"""
        self._tools[tool.name] = tool

    async def execute(
        self,
        name: str,
        arguments: dict
    ) -> str:
        """执行工具"""
        tool = self._tools.get(name)
        if tool is None:
            return f"Error: Tool '{name}' not found"

        try:
            return await tool.run(arguments)
        except Exception as e:
            return f"Error: {str(e)}"

会话管理 ​

Session 数据结构 ​

python
@dataclass
class Session:
    """对话会话"""
    key: str  # "telegram:123456789"
    messages: list[dict]  # 对话历史
    created_at: datetime
    updated_at: datetime
    stopped: bool = False
    last_consolidated: int = 0  # 已合并的消息数

    def add_message(self, msg: dict):
        """添加消息"""
        self.messages.append(msg)
        self.updated_at = datetime.now()

    def get_history(self, max_messages: int = 100) -> list[dict]:
        """获取未合并的历史消息"""
        unconsolidated = self.messages[self.last_consolidated:]
        return unconsolidated[-max_messages:]

记忆合并 ​

python
async def _consolidate_session(self, session: Session):
    """
    记忆合并：当对话过长时，压缩旧对话

    流程：
    1. 提取旧对话（未合并部分）
    2. 调用 LLM 压缩为摘要
    3. 更新 MEMORY.md（重要信息）
    4. 追加到 HISTORY.md（历史日志）
    5. 更新 last_consolidated 指针
    """
    # 1. 获取旧消息
    old_messages = session.messages[:session.last_consolidated + 50]

    # 2. 调用 LLM 压缩
    summary = await self._summarize_conversation(old_messages)

    # 3. 更新记忆文件
    await self.memory_store.update(
        session=session,
        summary=summary,
        messages=old_messages
    )

    # 4. 更新指针
    session.last_consolidated += len(old_messages)
