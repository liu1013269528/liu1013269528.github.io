nanobot 配置流程 (onboard) ​

首次使用 nanobot 时的配置向导详解

版本: v1.0.0 | 代码版本: nanobot v0.1.4.post4 | 更新日期: 2026-03-12

命令概览 ​

用法 ​

bash
nanobot onboard

功能 ​

onboard 命令用于初始化 nanobot 的配置和工作空间，包括：

  1. 创建配置文件 ~/.nanobot/config.json
  2. 创建工作空间目录 ~/.nanobot/workspace/
  3. 同步模板文件到工作空间
  4. 显示下一步操作提示

生活类比 ​

想象你刚搬进新家，onboard 就像是"入住向导"：

┌─────────────────────────────────────────────┐
│              入住向导 (onboard)              │
├─────────────────────────────────────────────┤
│  1. 📝 填写入住表 (创建 config.json)        │
│  2. 🏠 分配房间 (创建 workspace/)           │
│  3. 📋 提供使用手册 (同步模板文件)          │
│  4. 📍 告诉你钥匙在哪 (下一步提示)          │
└─────────────────────────────────────────────┘

配置路径解析 ​

文件: nanobot/config/paths.py

python
def get_config_path() -> Path:
    """
    获取配置文件路径

    优先级：
    1. 环境变量 NANOBOT_CONFIG
    2. ~/.nanobot/config.json (默认)
    """
    if "NANOBOT_CONFIG" in os.environ:
        return Path(os.environ["NANOBOT_CONFIG"])
    return Path.home() / ".nanobot" / "config.json"

配置保存 ​

文件: nanobot/config/loader.py

python
def save_config(config: Config) -> None:
    """
    保存配置到文件

    - 创建目录（如果不存在）
    - 使用 json 格式保存
    - 支持 camelCase 和 snake_case 键
    """
    config_path = get_config_path()
    config_path.parent.mkdir(parents=True, exist_ok=True)

    with open(config_path, "w", encoding="utf-8") as f:
        json.dump(config.model_dump(mode="json"), f, indent=2)

配置文件详解 ​

默认配置结构 ​

文件: ~/.nanobot/config.json

json
{
  // ===== 基本信息 =====
  "model": "claude-sonnet-4-20250514",

  // ===== LLM 提供商配置 =====
  "providers": {
    "anthropic": {
      "apiKey": "sk-ant-..."
    },
    "openai": {
      "apiKey": "sk-..."
    }
  },

  // ===== 启用的通道 =====
  "enabled": ["telegram"],

  // ===== Telegram 配置 =====
  "telegram": {
    "token": "your-bot-token",
    "allowFrom": ["*"]
  },

  // ===== 工作空间配置 =====
  "workspace": "~/.nanobot/workspace",

  // ===== Agent 配置 =====
  "agents": {
    "defaults": {
      "model": "claude-sonnet-4-20250514",
      "temperature": 0.7
    }
  }
}

配置项说明 ​

1. 模型选择 ​

字段类型默认值说明
modelstring-默认 LLM 模型

支持的模型格式：

  • claude-3-5-sonnet-20241022 (Anthropic Claude)
  • gpt-4 (OpenAI GPT)
  • gemini-pro (Google Gemini)
  • 任何 LiteLLM 支持的模型

2. 提供商配置 (providers) ​

json
{
  "providers": {
    "anthropic": {
      "apiKey": "sk-ant-...",
      "baseUrl": "https://api.anthropic.com"  // 可选
    },
    "openai": {
      "apiKey": "sk-...",
      "organization": "org-..."  // 可选
    }
  }
}

3. 通道配置 (enabled) ​

json
{
  "enabled": ["telegram", "discord", "slack"]
}

可用通道：

  • telegram - Telegram Bot
  • discord - Discord Bot
  • feishu - 飞书
  • slack - Slack
  • whatsapp - WhatsApp
  • email - Email
  • qq - QQ
  • dingtalk - 钉钉
  • matrix - Matrix

4. 权限配置 (allowFrom) ​

json
{
  "telegram": {
    "token": "...",
    "allowFrom": ["123456789", "987654321"]  // 只允许这些用户
  }
}

特殊值：

  • ["*"] - 允许所有用户
  • [] 或不设置 - 拒绝所有用户（v0.1.4.post4+ 默认行为）

⚠️ 安全提示：生产环境建议设置具体的用户 ID，避免使用 ["*"]

5. 工作空间 (workspace) ​

json
{
  "workspace": "~/.nanobot/workspace"
}

工作空间结构：

~/.nanobot/workspace/
├── sessions/          # 会话历史文件
├── skills/            # 用户技能
├── memory/            # 记忆文件
│   ├── MEMORY.md      # 长期记忆
│   └── HISTORY.md     # 历史日志
├── AGENTS.md          # Agent 配置
├── SOUL.md            # AI 个性
├── USER.md            # 用户偏好
└── HEARTBEAT.md       # 定期任务

环境变量覆盖 ​

配置可以通过环境变量覆盖：

bash
# 格式：NANOBOT__<section>__<key>
export NANOBOT__model="gpt-4"
export NANOBOT__agents__defaults__temperature="0.5"
export NANOBOT__telegram__token="your-token"
export NANOBOT__providers__anthropic__apiKey="sk-ant-..."

优先级：环境变量 > 配置文件

常见问题 ​

Q1: onboard 后无法启动？ ​

A: 检查以下几点：

  1. API Key 是否正确
bash
# 查看配置
cat ~/.nanobot/config.json | grep apiKey
  1. 模型名称是否正确
bash
# 常见错误
❌ "claude-3.5-sonnet"  # 错误格式
✅ "claude-sonnet-4-20250514"  # 正确格式
  1. 权限配置
bash
# 确保不是空数组（会拒绝所有）
"allowFrom": ["*"]  # 或具体用户 ID

Q2: 如何重置配置？ ​

A: 有两种方式：

方式 1：重新运行 onboard

bash
nanobot onboard
# 选择 y = overwrite with defaults

方式 2：手动删除配置

bash
rm ~/.nanobot/config.json
nanobot onboard

Q3: 如何使用不同的配置文件？ ​

A: 使用 --config 参数

bash
# 使用指定配置文件
nanobot gateway --config /path/to/custom-config.json
nanobot agent -c /path/to/custom-config.json

# 或设置环境变量
export NANOBOT_CONFIG=/path/to/custom-config.json
nanobot gateway

Q4: 配置文件位置在哪里？ ​

A: 按优先级：

  1. 环境变量 NANOBOT_CONFIG
  2. ~/.nanobot/config.json（默认）

查看当前配置位置：

bash
python -c "from nanobot.config.paths import get_config_path; print(get_config_path())"

Q5: 如何更改工作空间位置？ ​

A: 编辑配置文件：

json
{
  "workspace": "/custom/path/to/workspace"
}

或使用 --workspace 参数：

bash
nanobot agent --workspace /custom/path

Q6: 空的 allowFrom 有什么影响？ ​

A: 从 v0.1.4.post4 开始：

json
{
  "telegram": {
    "allowFrom": []  // 拒绝所有访问！
  }
}

解决方法：

json
{
  "telegram": {
    "allowFrom": ["*"]  // 允许所有
    // 或
    "allowFrom": ["123456789"]  // 允许特定用户
  }
}

配置最佳实践 ​

1. 安全存储 API Key ​

❌ 不推荐：直接写入配置文件

json
{
  "providers": {
    "anthropic": {
      "apiKey": "sk-ant-..."  // 可能被 git 提交
    }
  }
}

✅ 推荐：使用环境变量

bash
# ~/.bashrc 或 ~/.zshrc
export NANOBOT__providers__anthropic__apiKey="sk-ant-..."

2. 多实例配置 ​

为不同用途创建不同配置：

bash
# Telegram 实例
~/.nanobot-telegram/config.json

# Discord 实例
~/.nanobot-discord/config.json

# 启动
nanobot gateway --config ~/.nanobot-telegram/config.json
nanobot gateway --config ~/.nanobot-discord/config.json

3. 工作空间分离 ​

json
{
  "workspace": "~/projects/my-bot/workspace"
}

好处：

  • 项目数据独立
  • 便于备份和迁移
  • 可以有多个独立的 AI 助手
