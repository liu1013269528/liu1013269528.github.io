会话管理器 (SessionManager) ​

nanobot 的对话历史管理系统

版本: v1.0.0 | 代码版本: nanobot v0.1.4.post4 | 更新日期: 2026-03-12

模块概述 ​

职责 ​

SessionManager 负责管理对话会话：

  1. 会话存储：持久化对话历史
  2. 会话检索：快速加载会话
  3. 内存缓存：减少磁盘 I/O
  4. 历史提取：为 LLM 提供上下文

设计模式 ​

1. 缓存模式 (Cache Pattern) ​

定义：在内存中缓存频繁访问的数据

在 SessionManager 中的应用：

python
class SessionManager:
    def __init__(self, workspace: Path):
        self._cache: dict[str, Session] = {}

    def get_or_create(self, key: str) -> Session:
        # 1. 先检查缓存
        if key in self._cache:
            return self._cache[key]

        # 2. 缓存未命中，从磁盘加载
        session = self._load(key) or Session(key)

        # 3. 加入缓存
        self._cache[key] = session
        return session

好处：

  • ✅ 减少磁盘 I/O
  • ✅ 提高访问速度
  • ✅ 降低延迟

2. 活动记录模式 (Active Record Pattern) ​

定义：对象既包含数据也包含数据访问逻辑

在 Session 中的应用：

python
@dataclass
class Session:
    """会话数据类"""

    # 数据字段
    key: str
    messages: list[dict]
    created_at: datetime
    updated_at: datetime
    metadata: dict
    last_consolidated: int

    # 业务方法
    def add_message(self, role: str, content: str, **kwargs):
        """添加消息"""
        msg = {
            "role": role,
            "content": content,
            "timestamp": datetime.now().isoformat(),
            **kwargs
        }
        self.messages.append(msg)
        self.updated_at = datetime.now()

    def get_history(self, max_messages: int = 500) -> list:
        """获取历史记录"""
        ...

核心组件 ​

Session 类 ​

python
@dataclass
class Session:
    """
    对话会话

    存储格式：JSONL（每行一个 JSON）

    重要：消息是只追加的，为了 LLM 缓存效率
    """
    key: str                          # 会话键（channel:chat_id）
    messages: list[dict]              # 消息列表
    created_at: datetime              # 创建时间
    updated_at: datetime              # 更新时间
    metadata: dict                    # 元数据
    last_consolidated: int            # 已合并的消息数

    def add_message(self, role: str, content: str, **kwargs):
        """添加消息到会话"""

    def get_history(self, max_messages: int = 500) -> list:
        """获取历史记录（用于 LLM）"""

    def clear(self):
        """清空会话"""

SessionManager 类 ​

python
class SessionManager:
    """会话管理器"""

    def __init__(self, workspace: Path):
        self.workspace = workspace
        self.sessions_dir = workspace / "sessions"
        self._cache: dict[str, Session] = {}

    def get_or_create(self, key: str) -> Session:
        """获取或创建会话"""

    def save(self, session: Session):
        """保存会话到磁盘"""

    def invalidate(self, key: str):
        """使缓存失效"""

代码详解 ​

创建会话 ​

python
manager = SessionManager(workspace)

# 获取或创建会话
session = manager.get_or_create("telegram:123456789")

# session 现在包含：
# - key: "telegram:123456789"
# - messages: []
# - created_at: 当前时间
# - updated_at: 当前时间
# - last_consolidated: 0

添加消息 ​

python
# 添加用户消息
session.add_message(
    role="user",
    content="你好，nanobot！"
)

# 添加助手消息
session.add_message(
    role="assistant",
    content="你好！有什么我可以帮助你的？"
)

# 添加工具调用消息
session.add_message(
    role="assistant",
    content="",
    tool_calls=[{
        "id": "call_123",
        "type": "function",
        "function": {
            "name": "read_file",
            "arguments": '{"path": "README.md"}'
        }
    }]
)

# 添加工具结果
session.add_message(
    role="tool",
    content="README.md 内容...",
    tool_call_id="call_123",
    name="read_file"
)

获取历史 ​

python
# 获取最近 100 条消息
history = session.get_history(max_messages=100)

# 返回格式：
[
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！"},
    {"role": "assistant", "content": "", "tool_calls": [...]},
    {"role": "tool", "content": "...", "tool_call_id": "...", "name": "read_file"}
]

# 特点：
# 1. 只返回未合并的消息（messages[last_consolidated:]）
# 2. 对齐到用户消息（避免孤立的 tool_result）
# 3. 只保留必要字段（role, content, tool_calls 等）

保存会话 ​

python
# 保存到磁盘
manager.save(session)

# 文件格式（JSONL）：
{"role":"user","content":"你好","timestamp":"2026-03-12T10:00:00"}
{"role":"assistant","content":"你好！","timestamp":"2026-03-12T10:00:01"}
{"_type":"metadata","created_at":"2026-03-12T10:00:00","last_consolidated":0,"metadata":{}}

元数据行 ​

json
{
  "_type": "metadata",
  "created_at": "2026-03-12T10:00:00",
  "last_consolidated": 0,
  "metadata": {}
}

字段说明：

  • _type: 标识这是元数据行
  • created_at: 会话创建时间
  • last_consolidated: 已合并的消息数（用于记忆压缩）
  • metadata: 额外的会话元数据

常见问题 ​

Q1: 为什么使用只追加模式？ ​

A: 为了 LLM 缓存效率：

修改式（每次都改）：
┌──────────────────────────────┐
│ 消息 1                       │
│ 消息 2                       │
│ 消息 3（新增，需要重写整个文件）│
└──────────────────────────────┘
↓ 缓存失效

追加式（只追加新消息）：
┌──────────────────────────────┐
│ 消息 1                       │
│ 消息 2                       │
└──────────────────────────────┘
↓ 缓存保持有效
┌──────────────────────────────┐
│ 消息 1                       │
│ 消息 2                       │
│ 消息 3（追加，不影响已有内容） │
└──────────────────────────────┘

好处：

  • LLM 可以缓存前面的消息
  • 减少重复处理
  • 提高性能

Q2: last_consolidated 是什么？ ​

A: 跟踪已合并到记忆文件的消息数：

python
# 假设会话有 100 条消息
session.messages = [...]  # 100 条
session.last_consolidated = 80

# 未合并的消息：
unconsolidated = session.messages[80:]  # 最后 20 条

# 当对话过长时：
# 1. 将前 80 条合并到 MEMORY.md
# 2. 更新 last_consolidated = 100
# 3. 下次只读取新的消息

Q3: 如何清空会话？ ​

A: 使用 /new 命令或手动清空：

python
# 方式 1: /new 命令
# 用户发送: /new
# Agent 会：1. 合并当前记忆 2. 清空会话

# 方式 2: 手动清空
session.clear()
manager.save(session)

# clear() 会：
# - 清空消息列表
# - 重置 last_consolidated = 0
# - 更新 updated_at

Q4: 如何迁移旧会话？ ​

A: SessionManager 自动迁移：

python
# 旧路径 (~/.nanobot/sessions/)
# 新路径 (~/.nanobot/workspace/sessions/)

# SessionManager 会：
# 1. 检查新路径是否存在文件
# 2. 如果不存在，检查旧路径
# 3. 如果旧路径有文件，自动迁移
# 4. 记录日志

Q5: 如何备份会话？ ​

A: 直接复制会话目录：

bash
# 备份所有会话
cp -r ~/.nanobot/workspace/sessions ~/backup/sessions-$(date +%Y%m%d)

# 备份单个会话
cp ~/.nanobot/workspace/sessions/telegram\:123.jsonl ~/backup/
