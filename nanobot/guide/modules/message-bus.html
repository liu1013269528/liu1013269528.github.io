消息总线 (MessageBus) ​

nanobot 的异步消息传递核心

版本: v1.0.0 | 代码版本: nanobot v0.1.4.post4 | 更新日期: 2026-03-12

模块概述 ​

职责 ​

MessageBus 是 nanobot 的消息传递中心，负责：

  1. 解耦通道和代理：Channels 和 AgentLoop 通过消息总线通信，互不依赖
  2. 异步消息传递：使用 asyncio.Queue 实现非阻塞的消息传递
  3. 消息缓冲：缓存待处理的消息，防止丢失

定位 ​

┌─────────────┐              ┌─────────────┐              ┌─────────────┐
│  Channels   │ ──入站消息──▶│ MessageBus  │ ──出站消息──▶│ AgentLoop   │
│  (生产者)    │              │  (缓冲区)   │              │  (消费者)    │
└─────────────┘              └─────────────┘              └─────────────┘
  • 入站队列 (inbound)：聊天平台 → Agent
  • 出站队列 (outbound)：Agent → 聊天平台

文件结构 ​

nanobot/bus/
├── __init__.py      # 包初始化
├── events.py        # 消息类型定义
└── queue.py         # MessageBus 实现

设计模式 ​

1. 消息总线模式 (Message Bus Pattern) ​

定义：使用消息队列解耦组件的架构模式

为什么选择：

  • ✅ Channel 和 Agent 互不依赖
  • ✅ 可以独立测试和维护
  • ✅ 易于添加新的 Channel 或处理逻辑

生活类比：

想象一个"快递中转站"：

寄件人 (Channel)
    │
    ▼
┌─────────────────┐
│  快递中转站      │  ← MessageBus
│  (包裹暂存)      │
└─────────────────┘
    │
    ▼
收件人 (Agent)

- 寄件人不需要知道收件人是谁
- 收件人不需要知道包裹从哪里来
- 中转站可以同时处理多个包裹

2. 生产者-消费者模式 (Producer-Consumer) ​

定义：生产者生成消息，消费者处理消息

在 nanobot 中的应用：

角色组件动作
生产者Channelspublish_inbound()
消费者AgentLoopconsume_inbound()
生产者AgentLooppublish_outbound()
消费者Channelsconsume_outbound()

好处：

  • ✅ 异步处理，不阻塞
  • ✅ 消息缓冲，平滑流量
  • ✅ 可以有多个生产者和消费者

核心组件 ​

MessageBus 类 ​

文件: nanobot/bus/queue.py

python
class MessageBus:
    """异步消息总线"""

    def __init__(self):
        # 入站队列：聊天平台 → Agent
        self.inbound: asyncio.Queue[InboundMessage] = asyncio.Queue()

        # 出站队列：Agent → 聊天平台
        self.outbound: asyncio.Queue[OutboundMessage] = asyncio.Queue()

核心方法：

方法参数返回值说明
publish_inbound()InboundMessageNone发布入站消息
consume_inbound()无InboundMessage消费入站消息（阻塞）
publish_outbound()OutboundMessageNone发布出站消息
consume_outbound()无OutboundMessage消费出站消息（阻塞）
inbound_size无int入站队列大小（属性）
outbound_size无int出站队列大小（属性）

InboundMessage 类 ​

文件: nanobot/bus/events.py

python
@dataclass
class InboundMessage:
    """从聊天平台接收的消息"""

    channel: str                # "telegram", "discord" 等
    sender_id: str              # 用户 ID
    chat_id: str                # 聊天 ID
    content: str                # 消息文本
    timestamp: datetime         # 时间戳
    media: list[str]            # 媒体文件路径
    metadata: dict[str, Any]    # 额外元数据
    session_key_override: str | None = None  # 覆盖会话密钥

    @property
    def session_key(self) -> str:
        """会话唯一标识"""
        return self.session_key_override or f"{self.channel}:{self.chat_id}"

OutboundMessage 类 ​

文件: nanobot/bus/events.py

python
@dataclass
class OutboundMessage:
    """发送到聊天平台的消息"""

    channel: str                # "telegram", "discord" 等
    chat_id: str                # 聊天 ID
    content: str                # 响应文本
    reply_to: str | None = None # 回复的消息 ID
    media: list[str]            # 要发送的媒体
    metadata: dict[str, Any]    # 额外元数据

代码详解 ​

创建消息总线 ​

python
# 1. 导入必要的模块
import asyncio
from nanobot.bus.queue import MessageBus
from nanobot.bus.events import InboundMessage, OutboundMessage

# 2. 创建消息总线实例
bus = MessageBus()

# 3. 现在有了两个队列：
#    - bus.inbound (入站)
#    - bus.outbound (出站)

发布入站消息 ​

场景：Channel 收到用户消息，发布到 inbound

python
async def handle_user_message():
    """
    Channel 收到用户消息后的处理
    """
    # 创建入站消息
    msg = InboundMessage(
        channel="telegram",
        sender_id="123456789",
        chat_id="123456789",
        content="你好，nanobot！",
        media=[],
        metadata={"message_id": "123"}
    )

    # 发布到消息总线
    await bus.publish_inbound(msg)

    # ✓ 消息现在在 inbound 队列中
    # ✓ AgentLoop 会从队列中取走并处理

消费入站消息 ​

场景：AgentLoop 从队列取消息并处理

python
async def agent_loop():
    """
    AgentLoop 的主循环
    """
    while True:
        # 从队列消费消息（阻塞等待）
        msg = await bus.consume_inbound()

        # 处理消息
        print(f"收到消息: {msg.content}")
        print(f"来自: {msg.channel}:{msg.sender_id}")

        # ... 调用 LLM，执行工具等

发布出站消息 ​

场景：AgentLoop 处理完消息，发布响应到 outbound

python
async def send_response():
    """
    AgentLoop 处理完消息后发送响应
    """
    # 创建出站消息
    response = OutboundMessage(
        channel="telegram",
        chat_id="123456789",
        content="你好！有什么我可以帮助你的？",
        reply_to="123"  # 可选：回复原消息
    )

    # 发布到消息总线
    await bus.publish_outbound(response)

    # ✓ 消息现在在 outbound 队列中
    # ✓ ChannelManager 会分发给对应的通道

场景：Channel 从队列取消息并发送

async def channel_sender():

使用示例 ​

完整的消息流 ​

python
import asyncio
from nanobot.bus.queue import MessageBus
from nanobot.bus.events import InboundMessage, OutboundMessage

async def main():
    # 1. 创建消息总线
    bus = MessageBus()

    # 2. 模拟 Channel 收到消息
    print("📨 用户发送: 你好")
    inbound_msg = InboundMessage(
        channel="telegram",
        sender_id="123",
        chat_id="123",
        content="你好"
    )
    await bus.publish_inbound(inbound_msg)

    # 3. AgentLoop 处理消息
    msg = await bus.consume_inbound()
    print(f"🤖 Agent 收到: {msg.content}")

    # 4. AgentLoop 生成响应
    print("📝 Agent 思考中...")
    await asyncio.sleep(1)  # 模拟 LLM 调用

    # 5. 发送响应
    outbound_msg = OutboundMessage(
        channel="telegram",
        chat_id="123",
        content="你好！有什么我可以帮助你的？"
    )
    await bus.publish_outbound(outbound_msg)
    print(f"✉️  Agent 响应: {outbound_msg.content}")

    # 6. Channel 发送响应
    response = await bus.consume_outbound()
    print(f"📤 用户收到: {response.content}")

asyncio.run(main())

输出：

🤖 Agent 收到: 你好
📝 Agent 思考中...
✉️  Agent 响应: 你好！有什么我可以帮助你的？
📤 用户收到: 你好！有什么我可以帮助你的？

常见问题 ​

Q1: 为什么使用 Queue 而不是直接调用？ ​

A: Queue 提供了解耦和缓冲：

python
# ❌ 不使用 Queue - 紧耦合
class Channel:
    async def on_message(self, msg):
        # 直接调用 Agent，紧耦合
        await agent.process(msg)

# ✅ 使用 Queue - 解耦
class Channel:
    async def on_message(self, msg):
        # 发布到队列，解耦
        await bus.publish_inbound(msg)

好处：

  • Channel 不需要知道 Agent 的存在
  • Agent 可以独立测试
  • 可以添加监控、日志等中间层

Q2: 消息会丢失吗？ ​

A: 不会，因为：

  1. Queue 是内存队列：消息在队列中不会被丢弃
  2. 异步处理：使用 await 确保操作完成
  3. 阻塞获取：consume_*() 会阻塞直到有消息
python
# 消息不会被丢弃
await bus.publish_inbound(msg)  # 放入队列
msg = await bus.consume_inbound()  # 等待取出

⚠️ 注意：如果进程崩溃，内存中的消息会丢失。nanobot 没有持久化队列。

Q3: 如何处理消息积压？ ​

A: 可以使用以下策略：

python
# 1. 监控队列大小
if bus.inbound_size > 100:
    logger.warning(f"积压: {bus.inbound_size} 条消息")

# 2. 使用多个消费者
async def consumer_1():
    while True:
        msg = await bus.consume_inbound()
        await process(msg)

async def consumer_2():
    while True:
        msg = await bus.consume_inbound()
        await process(msg)

# 并行运行
await asyncio.gather(consumer_1(), consumer_2())

# 3. 设置超时（可选）
try:
    msg = await asyncio.wait_for(
        bus.consume_inbound(),
        timeout=5.0
    )
except asyncio.TimeoutError:
    logger.warning("没有消息，超时")

Q4: InboundMessage 和 OutboundMessage 有什么区别？ ​

A: 它们代表不同方向的消息：

特性InboundMessageOutboundMessage
方向聊天平台 → AgentAgent → 聊天平台
包含发送者信息回复信息
用途输入输出
字段sender_id, chat_idchat_id, reply_to

Q5: session_key 是如何生成的？ ​

A: 默认使用 channel:chat_id 格式：

python
# 默认会话键
msg = InboundMessage(
    channel="telegram",
    chat_id="123456789",
    ...
)
print(msg.session_key)  # "telegram:123456789"

# 可以覆盖
msg.session_key_override = "custom:thread:abc"
print(msg.session_key)  # "custom:thread:abc"

用途：用于标识和隔离不同的对话会话。

Q6: 如何调试消息流？ ​

A: 添加日志记录：

python
# 在发布时记录
await bus.publish_inbound(msg)
logger.info(f"[Inbound] {msg.channel}:{msg.chat_id} - {msg.content}")

# 在消费时记录
msg = await bus.consume_inbound()
logger.info(f"[Consuming] Processing {msg.session_key}")

# 检查队列状态
logger.debug(f"Queue sizes: inbound={bus.inbound_size}, outbound={bus.outbound_size}")
