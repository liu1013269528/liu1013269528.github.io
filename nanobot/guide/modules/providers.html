LLM 提供商系统 (Providers) ​

统一的 LLM 提供商接口

版本: v1.0.0 | 代码版本: nanobot v0.1.4.post4 | 更新日期: 2026-03-12

模块概述 ​

职责 ​

Provider 系统负责：

  1. 统一接口：所有 LLM 提供商的统一 API
  2. 提供商注册：管理和发现可用的提供商
  3. 模型路由：根据模型名称选择提供商
  4. 配置管理：处理 API Key 和配置

支持的提供商 ​

    ),
}

模型路由 ​

python
def get_provider_name(model: str) -> str:
    """
    根据模型名称确定提供商

    优先级：
    1. 模型前缀匹配
    2. 默认 litellm
    """
    for name, spec in PROVIDERS.items():
        if model.startswith(spec.model_prefix):
            return name
    return "litellm"

LLMProvider 基类 ​

python
class LLMProvider(ABC):
    """LLM 提供商抽象基类"""

    @property
    @abstractmethod
    def name(self) -> str:
        """提供商名称"""
        pass

    @abstractmethod
    async def chat(
        self,
        messages: list[dict],
        tools: list[dict] | None = None,
        model: str | None = None,
        temperature: float = 0.7,
        max_tokens: int = 4096,
    ) -> ChatResponse:
        """
        聊天接口

        Args:
            messages: 对话消息列表
            tools: 工具定义（用于 function calling）
            model: 模型名称
            temperature: 温度参数
            max_tokens: 最大 token 数

        Returns:
            ChatResponse（content, tool_calls, finish_reason）
        """
        pass

    def get_default_model(self) -> str:
        """获取默认模型"""
        pass

ChatResponse ​

python
@dataclass
class ChatResponse:
    """LLM 响应"""
    content: str | None           # 文本内容
    tool_calls: list[ToolCall]    # 工具调用
    finish_reason: str            # 结束原因（stop, length, error）
    usage: dict | None            # token 使用情况
    reasoning_content: str | None # 推理内容（Claude）
    thinking_blocks: list | None  # 思考块

使用示例 ​

创建提供商 ​

python
from nanobot.providers.registry import create_provider

# 方式 1: 从配置创建
provider = create_provider(config)

# 方式 2: 手动创建
from nanobot.providers.anthropic_provider import AnthropicProvider
provider = AnthropicProvider(api_key="sk-ant-...")

调用 LLM ​

python
response = await provider.chat(
    messages=[
        {"role": "system", "content": "You are a helpful assistant"},
        {"role": "user", "content": "Hello!"}
    ],
    tools=tool_definitions,  # 可选
    model="claude-sonnet-4-20250514",
    temperature=0.7,
)

# 访问响应
print(response.content)  # "Hello! How can I help you?"
print(response.tool_calls)  # []
print(response.finish_reason)  # "stop"

添加新提供商 ​

步骤 1: 创建 Provider 类 ​

python
# nanobot/providers/my_provider.py
from nanobot.providers.base import LLMProvider

class MyProvider(LLMProvider):
    @property
    def name(self) -> str:
        return "myprovider"

    async def chat(self, messages, tools=None, **kwargs):
        # 调用提供商 API
        response = await my_api_call(messages, tools)
        return ChatResponse(
            content=response.text,
            tool_calls=response.tool_calls,
            finish_reason=response.stop_reason
        )

步骤 2: 注册提供商 ​

python
# nanobot/providers/registry.py
PROVIDERS["myprovider"] = ProviderSpec(
    name="myprovider",
    model_prefix="my-",
    config_key="my_provider",
    default_model="my-model",
)

步骤 3: 添加配置 ​

python
# nanobot/config/schema.py
class ProvidersConfig(BaseModel):
    my_provider: ProviderConfig | None = None

常见问题 ​

Q1: 如何切换模型？ ​

A: 修改 config.json 中的 model 字段

Q2: 如何使用自定义端点？ ​

A: 在提供商配置中设置 baseUrl

Q3: 支持 Azure OpenAI 吗？ ​

A: 支持，使用 azure/ 前缀
