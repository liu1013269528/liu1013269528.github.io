模块概述 ​

职责 ​

AgentLoop 是 nanobot 的核心处理引擎，负责：

  1. 接收消息：从 MessageBus 消费入站消息
  2. 构建上下文：组装历史、记忆、技能等信息
  3. 调用 LLM：与 AI 模型交互
  4. 执行工具：处理 LLM 的工具调用请求
  5. 保存会话：持久化对话历史
  6. 发送响应：发布出站消息到 MessageBus

设计模式 ​

1. 事件循环模式 (Event Loop Pattern) ​

定义：持续监听和处理事件的主循环

在 AgentLoop 中的应用：

python
async def run(self):
    """主循环"""
    self._running = True
    while self._running:
        # 1. 等待消息
        msg = await self.bus.consume_inbound()

        # 2. 分发处理
        task = asyncio.create_task(self._dispatch(msg))

        # 3. 继续监听（不阻塞）

好处：

  • ✅ 持续运行，响应事件
  • ✅ 异步处理，不阻塞
  • ✅ 可以处理多个并发消息

2. 状态机模式 (State Machine) ​

定义：通过状态转换控制执行流程

在 AgentLoop 中的应用：

接收消息 → 检查命令 → 处理消息 → 迭代循环 → 发送响应
    │          │          │          │          │
    ▼          ▼          ▼          ▼          ▼
  [初始]     [命令]     [处理]     [迭代]     [完成]
              │                      │
              ├─ /new → [新会话]      │
              ├─ /stop → [停止]      │
              └─ /help → [帮助]      │

3. 迭代器模式 (Iterator Pattern) ​

定义：通过多次迭代逐步完成任务

在 AgentLoop 中的应用：

python
while iteration < self.max_iterations:
    # 1. 调用 LLM
    response = await self.provider.chat(messages, tools)

    # 2. 检查是否有工具调用
    if response.has_tool_calls:
        # 执行工具
        for tool_call in response.tool_calls:
            result = await self.tools.execute(...)
            messages.append(tool_result)
        # 继续下一次迭代
    else:
        # 没有工具调用，完成
        final_content = response.content
        break

流程图：

迭代 1: 用户消息 → LLM → 工具调用 A, B
迭代 2: 工具结果 A, B → LLM → 工具调用 C
迭代 3: 工具结果 C → LLM → 最终回答

核心组件 ​

AgentLoop 类 ​

文件: nanobot/agent/loop.py

python
class AgentLoop:
    """
    代理循环 - nanobot 的核心处理引擎

    职责：
    1. 从 MessageBus 消费消息
    2. 构建上下文（历史、记忆、技能）
    3. 调用 LLM
    4. 执行工具调用
    5. 保存会话
    6. 发送响应
    """

    def __init__(
        self,
        bus: MessageBus,           # 消息总线
        provider: LLMProvider,     # LLM 提供商
        workspace: Path,           # 工作空间
        model: str | None = None,  # 模型名称
        max_iterations: int = 40,  # 最大迭代次数
        temperature: float = 0.1,  # 温度参数
        max_tokens: int = 4096,    # 最大 token 数
        memory_window: int = 100,  # 记忆窗口
        ...
    ):
        # 核心组件
        self.bus = bus
        self.provider = provider
        self.workspace = workspace

        # 辅助组件
        self.context = ContextBuilder(workspace)      # 上下文构建器
        self.sessions = SessionManager(workspace)     # 会话管理器
        self.tools = ToolRegistry()                  # 工具注册表
        self.subagents = SubagentManager(...)        # 子代理管理

        # 配置
        self.model = model or provider.get_default_model()
        self.max_iterations = max_iterations
        self.temperature = temperature
        self.max_tokens = max_tokens
        self.memory_window = memory_window

        # 注册默认工具
        self._register_default_tools()

代码详解 ​

1. 主循环 (run) ​

python
async def run(self) -> None:
    """
    主循环 - 持续运行直到收到停止信号

    流程：
    1. 连接 MCP 服务器
    2. 循环消费消息
    3. 为每个消息创建异步任务
    """
    self._running = True

    # 1. 连接 MCP 服务器（懒加载）
    await self._connect_mcp()

    logger.info("Agent loop started")

    # 2. 主循环
    while self._running:
        try:
            # 从消息总线获取消息（1秒超时）
            msg = await asyncio.wait_for(
                self.bus.consume_inbound(),
                timeout=1.0
            )
        except asyncio.TimeoutError:
            # 超时继续，允许检查 _running 标志
            continue

        # 3. 处理特殊命令
        if msg.content.strip().lower() == "/stop":
            await self._handle_stop(msg)
        else:
            # 4. 为每个消息创建独立任务
            #    这样可以并发处理多个消息
            task = asyncio.create_task(self._dispatch(msg))

            # 5. 追踪活动任务
            self._active_tasks.setdefault(
                msg.session_key, []
            ).append(task)

            # 6. 任务完成后清理
            task.add_done_callback(
                lambda t, k=msg.session_key: (
                    self._active_tasks.get(k, [])
                    and self._active_tasks[k].remove(t)
                    if t in self._active_tasks.get(k, [])
                    else None
                )
            )

关键点：

  • 使用 asyncio.wait_for 设置超时，允许定期检查 _running 标志
  • 为每个消息创建独立任务，实现并发处理
  • 追踪活动任务，支持 /stop 命令取消

2. 消息分发 (_dispatch) ​

python
async def _dispatch(self, msg: InboundMessage) -> None:
    """
    消息分发 - 在全局锁下处理消息

    使用全局锁确保同一时间只处理一条消息
    """
    async with self._processing_lock:
        try:
            # 处理消息
            response = await self._process_message(msg)

            # 发送响应
            if response is not None:
                await self.bus.publish_outbound(response)
            elif msg.channel == "cli":
                # CLI 模式需要空响应来结束提示
                await self.bus.publish_outbound(
                    OutboundMessage(
                        channel=msg.channel,
                        chat_id=msg.chat_id,
                        content="",
                        metadata=msg.metadata or {},
                    )
                )

        except asyncio.CancelledError:
            # 任务被取消（如 /stop 命令）
            logger.info(
                "Task cancelled for session {}",
                msg.session_key
            )
            raise

        except Exception:
            # 处理异常，发送友好的错误消息
            logger.exception(
                "Error processing message for session {}",
                msg.session_key
            )
            await self.bus.publish_outbound(
                OutboundMessage(
                    channel=msg.channel,
                    chat_id=msg.chat_id,
                    content="Sorry, I encountered an error.",
                )
            )

为什么使用全局锁？

  • 防止同一会话的消息并发处理
  • 避免会话状态冲突
  • 确保消息按顺序处理

3. 迭代循环 (_run_agent_loop) ​

python
async def _run_agent_loop(
    self,
    initial_messages: list[dict],
    on_progress: Callable[..., Awaitable[None]] | None = None,
) -> tuple[str | None, list[str], list[dict]]:
    """
    迭代循环 - LLM 调用和工具执行的核心

    返回：(最终内容, 使用的工具, 所有消息)
    """
    messages = initial_messages
    iteration = 0
    final_content = None
    tools_used: list[str] = []

    # 最多迭代 max_iterations 次
    while iteration < self.max_iterations:
        iteration += 1

        # 1. 调用 LLM
        response = await self.provider.chat(
            messages=messages,
            tools=self.tools.get_definitions(),  # 工具定义
            model=self.model,
            temperature=self.temperature,
            max_tokens=self.max_tokens,
        )

        # 2. 检查是否有工具调用
        if response.has_tool_calls:
            # 2.1 发送进度更新（思考过程）
            if on_progress:
                # 移除 <thinking> 块
                thought = self._strip_think(response.content)
                if thought:
                    await on_progress(thought)
                # 发送工具提示
                await on_progress(
                    self._tool_hint(response.tool_calls),
                    tool_hint=True
                )

            # 2.2 添加助手消息到历史
            tool_call_dicts = [
                {
                    "id": tc.id,
                    "type": "function",
                    "function": {
                        "name": tc.name,
                        "arguments": json.dumps(
                            tc.arguments,
                            ensure_ascii=False
                        )
                    }
                }
                for tc in response.tool_calls
            ]
            messages = self.context.add_assistant_message(
                messages,
                response.content,
                tool_call_dicts,
            )

            # 2.3 执行每个工具调用
            for tool_call in response.tool_calls:
                tools_used.append(tool_call.name)
                args_str = json.dumps(
                    tool_call.arguments,
                    ensure_ascii=False
                )
                logger.info(
                    "Tool call: {}({})",
                    tool_call.name,
                    args_str[:200]
                )

                # 执行工具
                result = await self.tools.execute(
                    tool_call.name,
                    tool_call.arguments
                )

                # 添加工具结果到消息历史
                messages = self.context.add_tool_result(
                    messages,
                    tool_call.id,
                    tool_call.name,
                    result
                )

        else:
            # 3. 没有工具调用，完成
            clean = self._strip_think(response.content)

            # 检查是否是错误响应
            if response.finish_reason == "error":
                logger.error(
                    "LLM returned error: {}",
                    (clean or "")[:200]
                )
                final_content = (
                    clean or
                    "Sorry, I encountered an error calling the AI model."
                )
                break

            # 添加助手消息
            messages = self.context.add_assistant_message(
                messages, clean
            )
            final_content = clean
            break

    # 4. 检查是否达到最大迭代次数
    if final_content is None and iteration >= self.max_iterations:
        logger.warning(
            "Max iterations ({}) reached",
            self.max_iterations
        )
        final_content = (
            f"I reached the maximum number of tool call iterations "
            f"({self.max_iterations}) without completing the task. "
            "You can try breaking the task into smaller steps."
        )

    return final_content, tools_used, messages

4. 工具注册 ​

python
def _register_default_tools(self) -> None:
    """
    注册默认工具集

    工具可以分为几类：
    1. 文件操作（read, write, edit, list）
    2. Shell 执行
    3. Web 工具（search, fetch）
    4. 消息发送
    5. 子代理
    6. 定时任务
    """
    # 1. 文件操作工具
    allowed_dir = (
        self.workspace
        if self.restrict_to_workspace
        else None
    )
    for cls in (
        ReadFileTool,
        WriteFileTool,
        EditFileTool,
        ListDirTool,
    ):
        self.tools.register(
            cls(
                workspace=self.workspace,
                allowed_dir=allowed_dir
            )
        )

    # 2. Shell 执行工具
    self.tools.register(
        ExecTool(
            working_dir=str(self.workspace),
            timeout=self.exec_config.timeout,
            restrict_to_workspace=self.restrict_to_workspace,
            path_append=self.exec_config.path_append,
        )
    )

    # 3. Web 工具
    self.tools.register(
        WebSearchTool(
            api_key=self.brave_api_key,
            proxy=self.web_proxy
        )
    )
    self.tools.register(
        WebFetchTool(proxy=self.web_proxy)
    )

    # 4. 消息发送工具
    self.tools.register(
        MessageTool(
            send_callback=self.bus.publish_outbound
        )
    )

    # 5. 子代理工具
    self.tools.register(
        SpawnTool(manager=self.subagents)
    )

    # 6. 定时任务工具
    if self.cron_service:
        self.tools.register(
            CronTool(self.cron_service)
        )

常见问题 ​

Q1: 为什么需要 max_iterations？ ​

A: 防止无限循环：

python
# 可能的无限循环场景
用户: "帮我搜索 A"
迭代1: LLM 调用 web_search("A")
迭代1: 工具返回: 结果
迭代2: LLM: "我再搜索一下相关内容 B"
迭代2: LLM 调用 web_search("B")
迭代2: 工具返回: 结果
迭代3: LLM: "我再搜索一下 C"
... 无限循环

# 使用 max_iterations 限制
if iteration >= self.max_iterations:
    return "达到了最大迭代次数，任务未完成"

Q2: 如何处理并发消息？ ​

A: 使用全局锁：

python
async with self._processing_lock:
    # 确保同一时间只处理一条消息
    response = await self._process_message(msg)

好处：

  • 避免会话状态冲突
  • 保证消息按顺序处理
  • 防止资源竞争

Q3: _strip_think 是做什么的？ ​

A: 移除 LLM 的思考块：

python
# 某些模型（如 Claude）会返回思考块
response = """
<thinking>
用户想要分析文件，我需要先读取文件内容...
</thinking>

我来帮你分析 README.md...
"""

# 移除思考块
clean = self._strip_think(response.content)
# "我来帮你分析 README.md..."

用途：

  • 隐藏内部思考过程
  • 只向用户展示最终回答
  • 减少消息长度

Q4: 进度更新是如何工作的？ ​

A: 通过 on_progress 回调：

python
async def _bus_progress(content: str, *, tool_hint: bool = False):
    """发送进度更新到消息总线"""
    meta = dict(msg.metadata or {})
    meta["_progress"] = True      # 标记为进度消息
    meta["_tool_hint"] = tool_hint # 标记为工具提示
    await self.bus.publish_outbound(
        OutboundMessage(
            channel=msg.channel,
            chat_id=msg.chat_id,
            content=content,
            metadata=meta,
        )
    )

# 在迭代循环中调用
if on_progress:
    await on_progress("正在读取文件...")  # 用户看到的
    await on_progress("read_file(...)", tool_hint=True)  # 工具提示
