nanobot 架构设计 ​

核心设计理念 ​

nanobot 的设计遵循以下核心原则：

1. 解耦 ​

通过消息总线（MessageBus）解耦聊天通道和代理核心，使得：

  • 新增聊天平台只需实现通道接口
  • 代理核心不需要关心具体平台
  • 各组件独立开发和测试

2. 异步 ​

全面使用 asyncio 实现异步架构：

  • 高并发处理能力
  • 非阻塞 I/O 操作
  • 优秀的资源利用率

3. 插件化 ​

采用插件式架构：

  • 通道插件：轻松接入新的聊天平台
  • 提供商插件：支持多种 LLM
  • 工具插件：扩展 AI 能力

4. 轻量 ​

保持代码简洁和依赖最小化：

  • 核心代码约 4000 行
  • 最小化外部依赖
  • 清晰的代码结构

分层架构 ​

表现层 (Presentation Layer) ​

  • 通道系统: 处理不同聊天平台的消息收发
  • CLI 界面: 命令行交互界面

业务层 (Business Layer) ​

  • AgentLoop: 核心业务逻辑
  • 工具系统: AI 能力扩展
  • 会话管理: 对话状态维护

数据层 (Data Layer) ​

  • SessionManager: 会话持久化
  • MemoryStore: 长期记忆存储
  • SkillsLoader: 技能文件加载

基础设施层 (Infrastructure Layer) ​

  • MessageBus: 消息路由
  • CronService: 定时任务调度
  • HeartbeatService: 定期任务触发

设计模式应用 ​

模式位置作用
观察者模式MessageBus事件驱动架构
策略模式ProviderRegistry多提供商支持
工厂模式ChannelManager通道创建
命令模式ToolRegistry工具执行
建造者模式ContextBuilder上下文组装
单例模式SessionManager会话管理

数据流 ​

用户消息 → Channel → MessageBus.inbound
    → AgentLoop → LLM + Tools → MessageBus.outbound
    → Channel → 用户响应

扩展点 ​

nanobot 在以下方面提供扩展点：

  1. 新增聊天通道: 继承 BaseChannel
  2. 新增 LLM 提供商: 在 ProviderRegistry 注册
  3. 新增工具: 在 ToolRegistry 注册
  4. 新增技能: 在 workspace/skills/ 添加 SKILL.md
  5. 新增服务: 扩展辅助服务层
