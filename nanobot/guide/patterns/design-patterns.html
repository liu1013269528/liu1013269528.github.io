创建型模式 ​

工厂模式 (Factory Pattern) ​

定义：创建对象的接口由子类决定。

在 nanobot 中的应用：

python
# Provider 工厂
def create_provider(config: Config) -> LLMProvider:
    """根据配置创建对应的 Provider"""
    model = config.model

    if model.startswith("claude-"):
        return AnthropicProvider(config.anthropic)
    elif model.startswith("gpt-"):
        return OpenAIProvider(config.openai)
    else:
        return LiteLLMProvider(config.litellm)

# Channel 工厂
class ChannelManager:
    def _create_channel(self, name: str) -> BaseChannel:
        if name == "telegram":
            return TelegramChannel(...)
        elif name == "discord":
            return DiscordChannel(...)

好处：

  • ✅ 解耦对象创建和使用
  • ✅ 易于添加新类型
  • ✅ 集中管理创建逻辑

单例模式 (Singleton Pattern) ​

定义：确保一个类只有一个实例。

在 nanobot 中的应用：

python
# SessionManager 使用缓存实现伪单例
class SessionManager:
    def __init__(self, workspace: Path):
        self._cache: dict[str, Session] = {}

    def get_or_create(self, key: str) -> Session:
        """每个 key 只有一个 Session 实例"""
        if key in self._cache:
            return self._cache[key]

        session = self._load(key) or Session(key)
        self._cache[key] = session
        return session

好处：

  • ✅ 节省资源
  • ✅ 保证一致性
  • ✅ 全局访问点

结构型模式 ​

适配器模式 (Adapter Pattern) ​

定义：将一个接口转换为另一个接口。

在 nanobot 中的应用：

python
# Channel 适配器
class BaseChannel(ABC):
    """统一接口"""

    @abstractmethod
    async def send(self, msg: OutboundMessage):
        pass

# Telegram 适配器
class TelegramChannel(BaseChannel):
    async def send(self, msg: OutboundMessage):
        # 将统一的 OutboundMessage 转换为 Telegram 格式
        await self.bot.send_message(
            chat_id=msg.chat_id,
            text=msg.content,
            parse_mode="Markdown"
        )

# Discord 适配器
class DiscordChannel(BaseChannel):
    async def send(self, msg: OutboundMessage):
        # 将统一的 OutboundMessage 转换为 Discord 格式
        await self.channel.send(msg.content)

好处：

  • ✅ 统一接口
  • ✅ 隐藏平台差异
  • ✅ 易于添加新平台

装饰器模式 (Decorator Pattern) ​

定义：动态地给对象添加额外功能。

在 nanobot 中的应用：

python
# Tool 执行装饰器
async def execute_with_logging(name: str, params: dict):
    """装饰工具执行，添加日志"""
    logger.info(f"执行工具: {name}({params})")

    try:
        result = await original_execute(name, params)
        logger.info(f"工具成功: {name}")
        return result
    except Exception as e:
        logger.error(f"工具失败: {name}: {e}")
        return f"Error: {str(e)}"

好处：

  • ✅ 不修改原代码
  • ✅ 灵活添加功能
  • ✅ 可组合多个装饰器

行为型模式 ​

命令模式 (Command Pattern) ​

定义：将请求封装为对象。

在 nanobot 中的应用：

python
# Tool 作为命令
class Tool(ABC):
    @abstractmethod
    async def execute(self, **kwargs) -> str:
        """执行命令"""
        pass

# 具体命令
class ReadFileTool(Tool):
    async def execute(self, path: str) -> str:
        """执行读取文件命令"""
        with open(path) as f:
            return f.read()

# 命令执行器
class ToolRegistry:
    async def execute(self, name: str, params: dict) -> str:
        tool = self.get(name)
        return await tool.execute(**params)

好处：

  • ✅ 请求封装
  • ✅ 可排队执行
  • ✅ 支持撤销/重做

策略模式 (Strategy Pattern) ​

定义：定义算法族，分别封装。

在 nanobot 中的应用：

python
# Provider 策略
class LLMProvider(ABC):
    @abstractmethod
    async def chat(self, messages, tools, **kwargs):
        """聊天策略"""
        pass

# 具体策略
class AnthropicProvider(LLMProvider):
    async def chat(self, messages, tools, **kwargs):
        # Anthropic 特定实现
        pass

class OpenAIProvider(LLMProvider):
    async def chat(self, messages, tools, **kwargs):
        # OpenAI 特定实现
        pass

# 使用策略
class AgentLoop:
    def __init__(self, provider: LLMProvider):
        self.provider = provider  # 可以是任何策略

好处：

  • ✅ 算法可替换
  • ✅ 避免条件语句
  • ✅ 易于扩展

观察者模式 (Observer Pattern) ​

定义：对象状态改变时通知所有依赖。

在 nanobot 中的应用：

python
# MessageBus 作为主题
class MessageBus:
    def __init__(self):
        self._subscribers = []

    async def publish_inbound(self, msg: InboundMessage):
        """通知所有订阅者"""
        await self.inbound.put(msg)

# AgentLoop 作为观察者
class AgentLoop:
    async def run(self):
        while True:
            msg = await self.bus.consume_inbound()
            await self._process(msg)

好处：

  • ✅ 松耦合
  • ✅ 广播通信
  • ✅ 动态订阅

构建器模式 (Builder Pattern) ​

定义：分步骤创建复杂对象。

在 nanobot 中的应用：

python
class ContextBuilder:
    """上下文构建器"""

    def build_system_prompt(self) -> str:
        """构建系统提示词"""
        parts = []

        # 步骤 1: 添加身份
        parts.append(self._get_identity())

        # 步骤 2: 添加 Bootstrap
        bootstrap = self._load_bootstrap_files()
        if bootstrap:
            parts.append(bootstrap)

        # 步骤 3: 添加记忆
        memory = self.memory.get_memory_context()
        if memory:
            parts.append(f"# Memory\n\n{memory}")

        # 步骤 4: 添加技能
        skills = self.skills.build_skills_summary()
        if skills:
            parts.append(f"# Skills\n\n{skills}")

        return "\n\n---\n\n".join(parts)

好处：

  • ✅ 分步骤构建
  • ✅ 过程清晰
  • ✅ 易于扩展
