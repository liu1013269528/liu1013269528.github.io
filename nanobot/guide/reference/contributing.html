nanobot 文档贡献指南 ​

如何为 nanobot 文档做出贡献

版本: v1.0.0 | 更新日期: 2026-03-12

📝 贡献流程 ​

1. 确定贡献类型 ​

  • 修正错误：文档中的错误、过时信息
  • 补充内容：添加缺失的解释、示例
  • 改进表达：使文档更清晰易懂
  • 新增文档：添加新的模块或功能文档

2. 创建分支 ​

bash
# 从 main 分支创建功能分支
git checkout -b docs/update-module-name

3. 编辑文档 ​

bash
# 编辑文档
vim docs/modules/module-name.md

4. 预览更改 ​

bash
# 本地预览（如果有预览工具）
# 或者直接在编辑器中查看 Markdown

5. 提交更改 ​

bash
git add docs/
git commit -m "docs: 更新模块文档说明"

6. 创建 PR ​

bash
git push origin docs/update-module-name
# 在 GitHub 上创建 Pull Request

📄 文档规范 ​

目录结构 ​

docs/
├── overview/          # 总体概览
├── flow-analysis/     # 全流程分析
├── modules/           # 核心模块
├── patterns/          # 设计模式
├── annotated-code/    # 注释代码
├── indices.md         # 索引
├── README.md          # 主导航
└── TERMINOLOGY.md     # 术语表

命名规范 ​

  • 文件名：使用小写字母和连字符（module-name.md）
  • 标题：使用中文（# 模块名称）
  • 代码块：指定语言（````python`）
  • 链接：使用相对路径（[text](path/to/file.md)）

Markdown 格式 ​

markdown
# 一级标题

## 二级标题

### 三级标题

**粗体** 和 *斜体*

- 列表项 1
- 列表项 2

`代码` 或 `代码`

> 引用块

---

代码示例 ​

每个代码示例应包含：

python
# 简短的说明
def example():
    """函数说明"""
    pass

输出：

示例输出

📋 文档检查清单 ​

在提交 PR 前，请检查：

内容质量 ​

  • [ ] 技术准确性：代码示例可运行，路径正确
  • [ ] 完整性：涵盖必要的内容
  • [ ] 清晰度：初学者能理解
  • [ ] 实用性：提供实际价值

格式规范 ​

  • [ ] Markdown 格式正确
  • [ ] 链接有效
  • [ ] 代码块指定语言
  • [ ] 表格格式正确

一致性 ​

  • [ ] 术语一致（参考 TERMINOLOGY.md）
  • [ ] 风格一致（参考 TEMPLATE.md）
  • [ ] 标题层级合理

交叉引用 ​

  • [ ] 相关文档已链接
  • [ ] 文件路径正确
  • [ ] 锚点链接有效

🔍 常见编辑任务 ​

更新代码示例 ​

  1. 确保代码可运行
  2. 添加输出示例
  3. 添加注释说明

添加新章节 ​

  1. 遵循现有章节结构
  2. 更新目录（TOC）
  3. 添加到索引（如适用）

修正错误 ​

  1. 明确说明错误
  2. 提供正确信息
  3. 在 commit message 中引用相关 Issue

📧 联系方式 ​

有问题或建议？

  • 提交 Issue
  • 讨论

✅ 模板 PR 描述 ​

markdown

- 添加了新代码示例
- 修正了 Y 错误

- [x] 技术准确性
- [x] 格式规范
- [x] 链接有效
- [x] 术语一致
