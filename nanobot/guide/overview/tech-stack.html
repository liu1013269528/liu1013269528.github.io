Python 版本 ​

要求 ​

toml
requires-python = ">=3.11"

支持版本：

  • ✅ Python 3.11
  • ✅ Python 3.12

为什么需要 3.11+？

特性用途
Self 类型类型注解中的自我引用
try/except*更好的异常处理
性能提升Python 3.11 比 3.10 快 60%
asyncio 改进异步编程更高效

核心依赖 ​

1. 框架和 CLI ​

依赖版本用途为什么选择
typer>=0.20.0CLI 框架简洁优雅，自动生成帮助信息
rich>=14.0.0终端美化彩色输出、进度条、表格
prompt-toolkit>=3.0.50交互式输入REPL 模式的命令行输入

typer 示例：

python
import typer

app = typer.Typer()

@app.command()
def hello(name: str = typer.Option(..., help="你的名字")):
    """打招呼命令"""
    typer.echo(f"你好，{name}！")

if __name__ == "__main__":
    app()

2. AI 和 LLM ​

依赖版本用途为什么选择

litellm 优势：

python
from litellm import completion

# 统一接口调用不同模型
response = completion(
    model="gpt-4",  # 或 claude-3, gemini-pro 等

)

pydantic 示例：

python
from pydantic import BaseModel, Field

class Config(BaseModel):
    model: str = Field(..., description="LLM 模型名称")
    temperature: float = Field(default=0.7, ge=0, le=1)

    class Config:
        extra = "forbid"  # 禁止额外字段

代码质量 ​

依赖版本用途
ruff>=0.1.0代码检查和格式化

ruff 配置（pyproject.toml）：

toml
[tool.ruff]
line-length = 100
target-version = "py311"

[tool.ruff.lint]
select = ["E", "F", "I", "N", "W"]
ignore = ["E501"]  # 忽略行长度

使用方法：

bash
# 检查代码
ruff check nanobot/

# 自动修复
ruff check --fix nanobot/

# 格式化代码
ruff format nanobot/

依赖版本策略 ​

版本锁定 ​

nanobot 使用 兼容版本范围 而非精确版本：

toml
# ✅ 推荐：允许兼容更新
"typer>=0.20.0,<1.0.0"

# ❌ 不推荐：锁定精确版本
"typer==0.20.0"

好处：

  • 获得bug修复
  • 保持向后兼容
  • 避免依赖冲突
