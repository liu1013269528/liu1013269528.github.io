nanobot VitePress 文档网站 ​

nanobot 完整学习指南的 VitePress 网站。

项目结构 ​

vitepress/
├── .vitepress/              # VitePress 配置
│   ├── config.ts           # 网站配置
│   └── cache/              # 构建缓存（自动生成）
├── guide/                  # 文档内容
│   ├── index.md           # 指南首页
│   ├── introduction.md    # 项目介绍
│   ├── architecture.md    # 架构设计
│   ├── quick-start.md     # 快速开始
│   ├── overview/          # 概览系列
│   ├── flow-analysis/     # 流程分析
│   ├── modules/           # 模块详解
│   ├── patterns/          # 设计模式
│   ├── annotated-code/   # 注释代码
│   └── reference/         # 参考文档
├── index.md              # 网站首页
├── package.json          # npm 配置
└── README.md             # 本文件

开发指南 ​

安装依赖 ​

bash
npm install

本地开发 ​

bash
npm run dev

访问 http://localhost:5173

构建静态网站 ​

bash
npm run build

构建产物在 .vitepress/dist/ 目录。

预览构建结果 ​

bash
npm run preview

与 docs/ 的关系 ​

本项目是 docs/ 目录的 VitePress 发布版本。

  • docs/: 文档源文件
  • vitepress/: VitePress 网站版本

维护流程：

  1. 更新 docs/ 中的文档
  2. 同步到 vitepress/guide/
  3. 更新内部链接（如需要）
  4. 测试和构建

配置说明 ​

导航配置 ​

在 .vitepress/config.ts 中配置：

  • 顶部导航（Navbar）
  • 侧边导航（Sidebar）
  • 搜索功能
  • 主题设置

自定义主题 ​

当前使用 VitePress 默认主题。如需自定义，在 .vitepress/theme/ 目录下添加自定义文件。

部署 ​

静态托管 ​

构建后的静态网站（.vitepress/dist/）可以部署到任何静态托管服务：

  • GitHub Pages
  • Netlify
  • Vercel
  • Cloudflare Pages

GitHub Actions ​

可以配置 GitHub Actions 自动部署：

  1. 推送到 main 分支
  2. 自动触发构建
  3. 部署到 GitHub Pages
